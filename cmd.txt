Команды, которые необходимо использовать для развертывания базы данных Postgres с помощью Dockerfile (создание образа, запуск контейнера, создание тома).

Dockerfile:
FROM postgres:latest						   - базовый образ этого контейнера, основан на поледней версии PostgreSQL
ENV POSTGRES_PASSWORD=1337					   - создание переменной среды с паролем пользователя
ENV POSTGRES_USER=joel						   - создание переменной среды с именем пользователя
ENV POSTGRES_DB=db_joel                                            - создание переменной среды с названием базы данных
COPY init_scripts/init.sql /docker-entrypoint-initdb.d/init.sql    - копирование скрипта init.sql внутрь контейнера в специальное месте, где скрипты автоматически исполненяются при первом запуске контейнера
VOLUME /d_data:/var/lib/postgresql/data				   - создание тома на хосте и монтирование его внутри контейнера, для сохранения данных между запусками контейнера

Команды терминала для запуска контейнера:
1. docker build -t d_img:latest . - Команда для создания Docker-образа из Dockerfile. 
Опция "-t" используется для указания тега (имени) образа, а через двоеточие указывается версия этого образа, либо "latest" по-умолчанию. 
Точка в конце команды указывает на то, что Dockerfile находится в данной директории.

2. docker run -d -p 5432:5432 --name d_cont d_img:latest -v /d_data:/var/lib/postgresql/data - Команда запускает контейнер на основе указанного образа.
Опция "-d" используется для запуска контейнера в фоновом режиме (detached mode), т.е. пользователь снова получит доступ к командной строке.
Опция "-p" используется для связки портов хоста и контейнера.
Опция "--name" используется для указания имени контейнера (d_cont).
Аргумент "d_img:latest" - тег образа (его имя), на основе которого будет создан контейнер.
Опция "-v /d_data:/var/lib/postgresql/data" указывает местоположение внутри контейнера для тома, где будут храниться данные между запусками контейнера.
